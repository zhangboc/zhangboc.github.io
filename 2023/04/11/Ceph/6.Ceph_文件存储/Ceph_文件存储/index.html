Ceph FS文件存储

Ceph FS文件存储时由NAS衍生过来的。

Ceph 文件系统或CephFS是一个 POSIX 兼容的文件系统，构建在 Ceph 的分布式对象存储RADOS之上。CephFS致力于为各种应用程序提供最先进、多用途、高可用性和高性能的文件存储，包括共享主目录、HPC 暂存空间和分布式工作流共享存储等传统用例。

CephFS 通过使用一些新颖的架构选择来实现这些目标。值得注意的是，文件元数据存储在与文件数据不同的 RADOS 池中，并通过可调整大小的元数据服务器集群或MDS提供服务，它可以扩展以支持更高吞吐量的元数据工作负载。文件系统的客户端可以直接访问 RADOS 来读写文件数据块。因此，工作负载可能会随着底层 RADOS 对象存储的大小线性扩展；也就是说，没有网关或代理为客户端调解数据 I/O。

对数据的访问通过 MDS 集群进行协调，MDS 集群充当由客户端和 MDS 共同维护的分布式元数据缓存状态的权威。每个 MDS 将元数据的变化聚合成一系列高效写入 RADOS 上的日志；MDS 没有在本地存储任何元数据状态。该模型允许在 POSIX 文件系统的上下文中在客户端之间进行连贯和快速的协作。

CephFS 因其新颖的设计和对文件系统研究的贡献而成为众多学术论文的主题。它是 Ceph 中最古老的存储接口，曾经是 RADOS的主要用例。现在它与另外两个存储接口一起组成了一个现代统一存储系统：RBD（Ceph 块设备）和 RGW（Ceph 对象存储网关）。

安装MDS服务

在三个节点上同时部署MDS

[root@node-1 my-cluster]# ceph-deploy mds create node-1 node-2 node-3

查看当前MDS状态，因为当前没有文件系统，所以3台主机都是备状态

[root@node-1 my-cluster]# 
 3 up:standby

创建Ceph FS文件系统

创建pool

[root@node-1 my-cluster]# ceph osd pool create cephfs_data 16 16
[root@node-1 my-cluster]# ceph osd pool create cephfs_metadata 16 16

查看使用帮助

[root@node-1 my-cluster]# ceph -h|grep fs |grep new 
fs new <fs_name> <metadata> <data> {--force} {--allow-dangerous-make new filesystem using named pools <metadata> and <data>

创建一个fs文件系统

[root@node-1 my-cluster]# ceph fs new cephfs-demo cephfs_metadata cephfs_data  
new fs with metadata pool 9 and data pool 8

查看fs文件系统

[root@node-1 my-cluster]# ceph fs ls 
name: cephfs-demo, metadata pool: cephfs_metadata, data pools: [cephfs_data ]

查看集群状态

[root@node-1 my-cluster]#  ceph -s  #可以看到MDS状态已经又一个active了

内核挂载

创建一个挂载点

[root@node-1 my-cluster]# mkdir /mnt/cephfs #创建一个挂载点

使用命令进行挂载

[root@node-1 my-cluster]# which mount.ceph #使用这个命令进行挂载
/usr/sbin/mount.ceph 
[root@node-1 my-cluster]# rpm -qf /usr/sbin/mount.ceph #安装ceph-common的时候添加的命令
ceph-common-14.2.22-0.el7.x86_64
[root@node-1 my-cluster]# mount -t ceph 192.168.187.201:6789:/ /mnt/cephfs/ -o name=admin #挂载

查看挂载

挂载

可以看到与内核态挂载的地点文件是共享的 

[root@node-1 my-cluster]# ceph-fuse -n client.admin -m 192.168.187.201:6789,192.168.187.202:6789,192.168.187.203:6789 /mnt/ceph-fuse/ #使用ceph-fuse进行挂载，-n 指定用户名 -m 指定mon节点，可指定1个或者多个，或者不指定，最后写上挂载地点
